## Development Constraints

### Plugin Hot-Reload Limitation
**IMPORTANT**: After making changes to the plugin code and running `npm run build`, OpenCode must be **restarted** for the plugin changes to take effect. The plugin is loaded at OpenCode startup and does not support hot-reload.

Steps to test plugin changes:
1. Make code changes
2. Run `npm run build`
3. **Restart OpenCode** (the host application)
4. Test the updated functionality

## MRP (Model Routine Protocol)

This project uses MRP to manage repeatable agent routines. Before writing a script from scratch, check `mrp list` for existing routines.

**When to use MRP:**
- You're about to run a build, test, lint, or deploy step -> check `mrp list` first
- You've run the same command 2+ times this session -> create a routine with `mrp create`
- A routine failed -> review the output, then judge with `mrp judge`
- You need to modify a routine's script -> edit the file directly, then `mrp edit <id> --commit`

**Quick reference:**
```
mrp list                        - see available routines
mrp run <id>                    - execute a routine
mrp judge <id> "<run_id>" --status success|failure|partial --reason "..."
mrp show <id>                   - view details and run history
mrp create --name <n> --goal "<g>" --non-goals "<ng>" --success-criteria "<id:text>"
mrp --help                      - all commands
```
